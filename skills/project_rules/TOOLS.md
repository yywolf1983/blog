# TOOLS.md - 本地配置

技能定义了*工具如何工作*。这个文件是用于*你的*特定细节——那些只属于你的设置。

## 这里有什么

比如：
- 项目本地路径配置
- 规则检查忽略目录
- 自定义规则扩展
- 环境特定配置
- 工具偏好设置
- 开发环境配置

## 示例配置

### 项目路径配置
- 项目根目录：`/Users/yy/aaa/blog`
- 技能目录：`/Users/yy/aaa/blog/skills`
- 配置文件：`/Users/yy/aaa/blog/website.yaml`
- 脚本目录：`/Users/yy/aaa/blog/scripts`（可选）

### 规则检查配置
- 忽略目录：`["node_modules", ".git", "build", "dist", "venv"]`
- 检查深度：`10`
- 检查文件类型：`[".md", ".yaml", ".yml", ".py"]`
- 自动修复：`false`（设置为true时会自动修复某些规则问题）

### 自定义规则配置
- 技能命名规范：`小写字母+下划线`
- 技能描述长度：`不超过200字`
- 技能参数数量：`不超过10个`
- 技能版本格式：`语义化版本（如1.0.0）`
- 技能分类：`["工具类", "数据类", "分析类", "其他"]`

### 环境配置
- 开发环境：`local`
- 调试模式：`true`
- 日志级别：`info`（可选值：debug, info, warning, error）
- 编辑器：`Visual Studio Code`

### 脚本配置
- 生成索引脚本：`generate_readme.py`
- 脚本执行权限：`true`
- 自动执行：`false`（设置为true时会在特定事件触发脚本执行）

### 备份配置
- 自动备份：`true`
- 备份频率：`daily`（可选值：hourly, daily, weekly）
- 备份路径：`/Users/yy/aaa/blog/backups`
- 备份保留天数：`30`

## 环境变量配置

如果需要使用环境变量，可以在项目根目录创建`.env`文件（添加到.gitignore）：

```
# 项目路径
PROJECT_ROOT=/Users/yy/aaa/blog
SKILLS_DIR=/Users/yy/aaa/blog/skills

# 编辑器设置
EDITOR=code

# 备份设置
BACKUP_PATH=/Users/yy/aaa/blog/backups
```

## 配置加载顺序

1. 环境变量（最高优先级）
2. `.env`文件
3. 此TOOLS.md文件
4. 技能默认配置（最低优先级）

## 为什么要分开？

技能是共享的。你的设置是你的。将它们分开意味着你可以：
- 更新技能而不丢失你的本地配置
- 分享技能而不泄露你的基础设施信息
- 在不同环境中使用不同的配置
- 保持技能定义的简洁性和通用性

## 如何使用此配置

1. 根据你的本地环境修改上述配置值
2. 保存文件，配置会在技能执行时自动加载
3. 如需添加新的配置项，直接在相应部分添加即可
4. 对于敏感信息（如API密钥），建议使用环境变量或.env文件
