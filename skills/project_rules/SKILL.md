# 技能名称：项目规则技能

## 描述
这是一个项目规则技能，用于记录和管理项目的规则和规范，确保项目的一致性、可维护性和可扩展性。

## 功能
- 功能1：记录项目的目录结构规范
- 功能2：管理文件命名和内容规范
- 功能3：维护技能系统的使用规则
- 功能4：提供项目配置和导航管理指南

## 参数
- `rule_type`：字符串类型，规则类型（如目录结构、文件命名、技能管理等）
- `scope`：字符串类型，规则适用范围（默认值：整个项目）
- `level`：字符串类型，规则重要程度（默认值：普通）

## 使用示例
```python
# 示例代码
result = get_project_rules(rule_type="目录结构", scope="skills/", level="重要")
print(result)
```

## 项目规则详细说明

### 1. 目录结构规则
- 项目以目录结构维护，所有内容按功能分类组织
- 每个目录必须包含 `index.md` 文件，用于生成文件列表
- 技能目录（`skills/`）包含多个技能子目录，每个子目录代表一个技能
- 技能子目录应包含 `SKILL.md` 和 `TOOLS.md` 文件

### 2. 文件生成规则
- `index.md` 文件由 `generate_readme.py` 脚本自动生成
- 不要手动修改 `index.md` 文件，脚本会随时覆盖
- 生成的 `index.md` 文件只包含文件列表，不包含任何笔记内容

### 3. 技能系统规则
- 在 `skills/` 目录中添加新技能时，创建独立的技能子目录
- 技能子目录应包含：
  - `SKILL.md`：技能定义文件，包含技能的功能、参数、使用示例等
  - `TOOLS.md`：本地配置文件，用于存储特定于环境的设置
- 不要在技能目录中添加笔记内容，保持目录结构简洁

### 4. 网站配置规则
- 目录结构发生变更时，需要检查 `website.yaml` 配置文件
- 若添加新的技能目录，需要在 `website.yaml` 中更新相应的导航配置
- 导航配置应包含 `title`、`icon`、`link` 等字段

### 5. 版本控制规则
- 请勿私自提交任何更改，如有需要请先与项目维护者沟通
- 提交更改时，使用清晰、描述性的提交信息
- 避免提交敏感信息，如API密钥、密码等

### 6. 提醒机制
- 目录结构变更后，应提醒检查 `website.yaml` 配置
- 技能添加或修改后，应提醒运行 `generate_readme.py` 脚本更新索引文件
- 任何重大变更应先与项目维护者沟通，获得批准后再执行
